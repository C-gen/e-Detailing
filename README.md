# e-Detailing

HTML5 e-Detailing TODO list

#### TODO-лист для программирования презентации для HTML5 платформы

В репозитории лежит пример готовой презентации. Слайд Title_Brand_RU - пустой. Слайд Popup_Brand_RU с примером попап-окна. Слайд Video_Brand_RU с пример встроенного видео. Слайд Animation_Brand_RU с примером css анимации.

Для начала прописываем основные свойства всей презентации в общем файле ```common/scripts/scripts_base.js```. Так же учтем, что папка common является общей для всех слайдов.

1. Создаем карту презентации в виде JSON в переменной map. Для описания вертикальных презентаций достаточно указать родительские презентацию и слайд в переменных "p_pres" и "p_slide". Именование презентаций и слайдов должно выглядеть так: "SlideName_Brand_RU" для слайдов и "PresentationName_Brand_RU" для презентаций, где строки "SlideName" и "PresentationName" произвольные.
2. Указываем генерировать ли нам кастомное меню и с какими элементами, где el_class - класс элемента, show - активен или нет, кроме слайдов указанных в except.
3. Далее в links указываем переходы по ссылкам. Тут рамзещаем все ссылки, включая те, что созданы в предыдущем пункте. Ссылки привязываются по классам.
4. Далее в переменной show_bc указываем генерировать ли хлебные крошки.
5. Переменные goto_back_btn и single_link можно не менять и использовать их по назначению. Создание ссылок через "single_link" не рекомендуется, так как при этом описание действий ссылок не будет централизованным.
6. Далее в функции set_events прописываем все остальные события, имеющиеся на всех слайдах (например запуск анимации или открытия подсказки).

После создания общих свойств, переходим к созданию каждого слайда. 

1. Переименовываем папку Title_Brand_RU, внутрилежащий html и миниатюры в то имя, которое соответствует первому слайду первой презентации. То есть имена папки, html файла и миниатюр должны совпадать, только у миниатюр будут постфиксы -full и -thumb. 
2. В теге ```<title>``` прописываем название первой презентации, для того, чтобы при просмотре презентации на компьютере скрипт знал к какой презентации относится данный слайд. Для остальных слайдов эту информацию прописывать не обязательно. 
2. На основе созданного первого слайда делаем его копии и переименовываем их в те названия слайдов, которые мы определили в карте презентации.
3. Далее в каждую папку со слайдом копируем общую папку common.
4. После этого в каждой папке со слайдом, если необходимо пишем свои стили и скрипты в файлах scripts.js и style.css, и тут же в папку images помещаем необходимые изображения. В html весь контент пишется в классе ```.content```.
5. Презентация готова. Запаковываем каждый слайд в архив zip и отправляем в админку.
